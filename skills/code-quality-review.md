# Code Quality Review

## Description
A systematic approach to reviewing code for quality, maintainability, and best practices.

## When to Use
- Before merging pull requests
- During code reviews
- Before releasing new features
- For refactoring sessions

## Example Usage
```bash
# Use Claude Code to review your code
claude review src/components/UserProfile.js
```

## Benefits
- ✅ Catch bugs early
- ✅ Ensure code follows best practices
- ✅ Improve code readability
- ✅ Reduce technical debt
- ✅ Share knowledge among team members

## Tips
- Start with high-level architecture
- Check for common security vulnerabilities
- Verify error handling is complete
- Ensure test coverage is adequate
- Look for code duplication
- Check for performance issues
- Verify naming conventions
- Ensure comments are meaningful
- Review dependencies
- Check for TODOs and FIXMEs
