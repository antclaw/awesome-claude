# Refactoring

## Description
Improve code structure, readability, and maintainability without changing functionality.

## When to Use
- When code becomes hard to understand
- Before adding new features
- During code reviews
- Regularly to prevent technical debt

## Example Usage
```bash
# Refactor a file
claude refactor src/components/UserList.js
```

## Benefits
- ✅ Easier to understand
- ✅ Easier to maintain
- ✅ Easier to extend
- ✅ Fewer bugs
- ✅ Better performance
- ✅ Improved readability

## Tips
- Refactor in small steps
- Run tests after each change
- Make sure tests still pass
- Focus on one thing at a time
- Rename variables and functions
- Extract functions for clarity
- Remove code duplication
- Simplify complex conditions
- Improve function and class names
- Add missing error handling
- Remove commented-out code
- Follow SOLID principles
- Keep the same functionality
