# Debugging Strategy

## Description
A systematic approach to finding and fixing bugs efficiently.

## When to Use
- When encountering unexpected errors
- When tests are failing
- When behavior doesn't match expectations
- During troubleshooting sessions

## Example Usage
```bash
# Use Claude Code to help debug
claude debug src/api/users.js
```

## Benefits
- ✅ Faster bug resolution
- ✅ More reliable fixes
- ✅ Reduced debugging time
- ✅ Better understanding of issues
- ✅ Prevention of similar bugs

## Tips
1. **Reproduce the issue** - Create a minimal reproducible case
2. **Read error messages** - Understand what went wrong
3. **Check recent changes** - Look at what changed recently
4. **Add logging** - Insert strategic log statements
5. **Isolate the problem** - Narrow down the scope
6. **Use debugging tools** - Debuggers, breakpoints, console logs
7. **Test incrementally** - Fix one thing at a time
8. **Review similar issues** - Check if this is a known problem
9. **Ask for help** - Don't debug alone
10. **Document the fix** - Prevent future issues
