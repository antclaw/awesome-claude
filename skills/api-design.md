# API Design

## Description
Create well-designed, RESTful, and GraphQL APIs that are easy to use and maintain.

## When to Use
- Designing new APIs
- Refactoring existing APIs
- Planning API changes
- Creating API documentation

## Example Usage
```bash
# Design a new API endpoint
claude design-api POST /api/users
```

## Benefits
- ✅ Consistent API design
- ✅ Better developer experience
- ✅ Easier to maintain
- ✅ Clear documentation
- ✅ Scalable architecture

## Tips
- Follow RESTful principles
- Use meaningful HTTP methods (GET, POST, PUT, DELETE)
- Use proper status codes
- Implement proper error handling
- Add API versioning
- Document with OpenAPI/Swagger
- Consider rate limiting
- Implement authentication
- Use consistent naming conventions
- Provide example requests and responses
- Make APIs idempotent where possible
