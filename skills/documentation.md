# Documentation Writing

## Description
Create clear, comprehensive, and maintainable documentation for code, APIs, and systems.

## When to Use
- Starting a new project
- Adding new features
- Refactoring code
- Creating API documentation
- Onboarding new team members
- Before releasing software

## Example Usage
```bash
# Generate documentation for code
claude document src/app.js

# Create API documentation
claude document-api src/routes/api.js
```

## Benefits
- ✅ Easier onboarding
- ✅ Better collaboration
- ✅ Reduced support burden
- ✅ Self-documenting code
- ✅ Clearer communication

## Tips
- Start with README.md
- Document code with comments
- Create API documentation (Swagger/OpenAPI)
- Write user guides and tutorials
- Include examples and use cases
- Keep documentation up to date
- Use clear and concise language
- Include diagrams and flowcharts
- Document architecture and design decisions
- Provide troubleshooting guides
- Include screenshots and videos
- Use versioning for documentation
- Make documentation searchable
- Link to external resources
- Document dependencies and requirements
- Include contribution guidelines
- Use Markdown for formatting
- Consider interactive documentation (Storybook, GitBook)
- Regularly review and update documentation
