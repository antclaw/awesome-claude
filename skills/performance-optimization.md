# Performance Optimization

## Description
Analyze and optimize code, database queries, and infrastructure for better performance and faster execution.

## When to Use
- Code is running slowly
- High latency in requests
- Database performance issues
- Before scaling infrastructure
- After major feature additions

## Example Usage
```bash
# Analyze performance bottlenecks
claude performance-optimize src/app.js

# Optimize database queries
claude performance-optimize db/query.js
```

## Benefits
- ✅ Faster response times
- ✅ Better user experience
- ✅ Reduced server costs
- ✅ Higher throughput
- ✅ Improved scalability

## Tips
- Profile code to find bottlenecks
- Optimize database queries (add indexes, use proper joins)
- Implement caching strategies (Redis, CDN, browser caching)
- Code splitting and lazy loading
- Optimize images and assets
- Use efficient algorithms and data structures
- Minify and bundle code
- Implement pagination for large datasets
- Use connection pooling
- Optimize memory usage
- Implement proper logging and monitoring
- Use async/await for I/O operations
- Avoid blocking operations
- Implement rate limiting
- Use Web Workers for CPU-intensive tasks
- Optimize CSS and JavaScript
- Use HTTP/2 or HTTP/3
- Implement content delivery network (CDN)
- Monitor performance metrics (Google Lighthouse, New Relic)
- Use APM tools (DataDog, New Relic)
